==============================================================================
        GNU開発環境・インストール手順(Linux)
==============================================================================
--------------------------------------------------
1. ディレクトリの作成
--------------------------------------------------

開発環境をインストールするディレクトリを作成します。この手順書では、/usr/local
/tef_em1d にインストールするものとします。

  例: % mkdir /usr/local/tef_em1d

  ※ 他のT-Kernel用GNU開発環境が既にインストールされている場合は、それとは異な
     るディレクトリに本開発環境をインストールしてください。 同じディレクトリに
     インストールした場合は、トラブルを生じる可能性があります。

  ※ 通常、/usr/local の所有者は root となっているため、/usr/local/tef_em1d を
     作成するためには root 権限が必要になります。/usr/local/tef_em1d の作成後、
     その所有者およびパーミッションは、T-Kernel の開発作業を行うユーザがアクセ
     スできるよう適切に設定してください。

--------------------------------------------------
2. 開発環境パッケージの展開
--------------------------------------------------

/usr/local/tef_em1d 内で、GNU 開発環境パッケージを展開します。

  例: % cd /usr/local/tef_em1d
      % tar zxpf te.Linux-i686.arm_2.1.0.1.tar.gz
      % tar zxpf te.Linux-i686.common.12.tar.gz

パッケージの展開後は、上記の *.tar.gz のファイルは不要ですので、削除してくださ
い。

--------------------------------------------------
3. 開発用ツールの準備
--------------------------------------------------

開発環境を使用するために、以下のツールが必要です。システムにインストールされて
いない場合は、適当な手段で入手し、利用可能な状態にしておいてください。

        GNU Make
        Perl

Perlは、/usr/local/bin/perl によって起動するようにしてください。このため、/usr
/local/bin から /usr/bin/perl にシンボリックリンクを張ってください。

  例: % cd /usr/local/bin
      % ln -s /usr/bin/perl

--------------------------------------------------
4. 環境変数GNU_BD,GNUarm_2の設定
--------------------------------------------------

T-Kernel のビルドを行うために、環境変数 GNU_BD, GNUARM_2 を設定します。

  例: % export GNU_BD=/usr/local/tef_em1d/tool/Linux-i686
      % export GNUARM_2=$GNU_BD/arm_2-unknown-tkernel

--------------------------------------------------
5. 環境変数BDの設定
--------------------------------------------------

環境変数 BD に、T-Kernelソースのトップディレクトリを設定します。

  例: % export BD=/usr/local/tef_em1d/tkernel_source
